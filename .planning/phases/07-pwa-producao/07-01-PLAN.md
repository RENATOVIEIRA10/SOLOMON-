<objective>
Tornar o PWA funcional EM PRODUÇÃO. Estado atual: manifest.json (standalone, maskable),
ícones 16-512 + apple-icon, metas appleWebApp e rota /~offline já existem e funcionam;

`@serwist/turbopack` por completo.
</objective>

<task id="1">
<name>public/sw.js vanilla + registro próprio</name>
<read_first>
- app/src/sw.ts (comportamento a replicar: skipWaiting, clientsClaim, fallback /~offline para documents)
- app/src/app/layout.tsx (onde o SerwistProvider envolve children — será substituído)
- app/src/app/~offline/page.tsx (rota de fallback — só ler, não alterar)
- app/next.config.ts (remover withSerwist)
- app/package.json (remover @serwist/turbopack)
</read_first>
<action>
1. Criar `app/public/sw.js` (JS puro, sem imports) com:
   - `const CACHE_VERSION = 'solomon-v1'` — bump manual em mudanças de estratégia.
   - install: `caches.open(CACHE_VERSION)` pré-cacheando EXATAMENTE `['/~offline', '/manifest.json', '/icon-192.png', '/icon-512.png', '/solomon-wordmark.png']`; `self.skipWaiting()`.
   - activate: deletar caches cujo nome !== CACHE_VERSION; `self.clients.claim()`.
   - fetch:
     a) `request.method !== 'GET'` → return (não interceptar).
     b) `url.pathname.startsWith('/api/')` → return (NUNCA cachear API — respostas do oráculo são dinâmicas e autenticadas).
     c) `request.destination === 'document'` → network-first: `fetch(request)` com catch → `caches.match('/~offline')`.
     d) destinations `style|script|font|image` → cache-first com revalidação: tenta cache, senão fetch e `cache.put` (só respostas 200 e mesma origem).
2. Criar `app/src/components/sw-register.tsx`:
   - `"use client"`; useEffect que roda `navigator.serviceWorker.register('/sw.js')` se suportado e `process.env.NODE_ENV === 'production'` (em dev não registra — evita cache atrapalhando next dev).
   - Listener de `controllerchange` que NÃO força reload (skipWaiting já ativa o worker novo; reload forçado no meio de uma resposta do oráculo seria hostil).
   - Render `null`.
3. `app/src/app/layout.tsx`: remover import/uso de `SerwistProvider` (children ficam diretos no body) e adicionar `<SwRegister />` dentro do body.
4. `app/next.config.ts`: remover `withSerwist` — exportar `nextConfig` puro.
5. Deletar `app/src/sw.ts` e `app/src/app/serwist.ts`.
6. `cd app && npm uninstall @serwist/turbopack`.
</action>
